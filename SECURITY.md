# Security Policy

## Supported versions

The latest published minor version receives security fixes.

## Reporting a vulnerability

Please do not open a public issue for suspected vulnerabilities. Email the maintainer or use GitHub private vulnerability reporting when enabled.

Include:

- affected version or commit;
- operating system and platform backend;
- proof-of-concept steps;
- expected and actual impact.

We aim to acknowledge reports within 7 days and coordinate a fix before disclosure.
